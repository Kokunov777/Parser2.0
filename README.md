**Ограничения: Любые особенности реализации, известные ограничения.**
Строгая грамматика: Анализатор распознает только одну конкретную конструкцию:
let <идентификатор> = num::complex::Complex::new(<число>, <число>);

Любое отклонение от этого шаблона (пропуск ;, опечатка в complex, лишние пробелы в ключевых местах) будет расценено как ошибка.
Однострочный анализ: Анализатор предназначен для проверки одной строки кода. Многострочные конструкции не поддерживаются.

Типы чисел: Вещественные числа могут быть записаны как с целой, так и с дробной частью (3.1, -4.2). Экспоненциальная форма (1e5) не поддерживается.
Метод восстановления: Используется метод нейтрализации ошибок Айронса. При обнаружении первой ошибки анализатор пытается "пропустить" некорректные токены до ближайшего восстанавливающего символа (например, ;), после чего может продолжить анализ и найти другие ошибки.

Отсутствие подсветки синтаксиса: Встроенный редактор (QTextEdit) не имеет динамической подсветки синтаксиса Rust. Подсветка применяется только для указания местоположения ошибки после клика по таблице.
Зависимость от PyQt6: Для работы приложения необходима библиотека PyQt6. Убедитесь, что она установлена корректно.

Грамматика G[Z] для объявления комплексного числа на языке Rust является автоматной (тип 3) и относится к подклассу праволинейных автоматных грамматик. Это означает, что для неё существует детерминированный конечный автомат, который может быть эффективно реализован в виде программы-распознавателя.


**Схема автоматов**
<img width="782" height="1107" alt="grammatic drawio" src="https://github.com/user-attachments/assets/a17c906c-f57a-4070-ac22-7525c05ad74a" />


**Классификация грамматики (по Хомскому).**
Согласно определению (см. раздел 2.7), правила автоматной грамматики должны иметь вид A → aB или A → a, где A, B ∈ V_N, a ∈ V_T.

Правило 1 (<Объявление> → let id : path = <Инициализатор> ;) не удовлетворяет этому условию, так как в правой части содержит несколько терминалов, за которыми следует нетерминал.

Правило 5 (<ВызовКонструктора> → path ( float , float )) также не соответствует, так как в нем отсутствует нетерминал-преемник в конце цепочки.
Вывод: грамматика не является автоматной.

Проверка на принадлежность к контекстно-свободным (КС) грамматикам.
Правила КС-грамматики имеют вид A → α, где A ∈ V_N, а α ∈ (V_T ∪ V_N)*.

Все правила грамматики G[<Объявление>] удовлетворяют этому критерию. В левой части каждого правила находится ровно один нетерминальный символ (<Объявление>, <Инициализатор>, <ВызовКонструктора>).
Вывод: грамматика является контекстно-свободной (КС-грамматикой).

Проверка на принадлежность к контекстно-зависимым (КЗ) грамматикам и грамматикам типа 0.
Поскольку любой КС-язык является также и КЗ-языком, наша грамматика формально принадлежит и к типу 1. Однако практическая ценность этого факта невелика, так как для реализации анализатора требуется более узкий, эффективно разбираемый подкласс.

*Реализация*

**Регулярные выражения**

` 1. Годы 2000–2010 `

**Описание задачи:** Найти все вхождения годов от 2000 до 2010 включительно, которые не являются частью более длинных чисел.

**Регулярное выражение:**
```
\b(200[0-9]|2010)\b
```

**Пояснение обозначений:**
- `\b` – граница слова, гарантирует, что год не является частью большего числа.
- `200[0-9]` – соответствует 2000–2009.
- `2010` – соответствует 2010.
- `|` – логическое ИЛИ.

**Примеры строк, которые должны находиться:**
- `2000`
- `2005`
- `2009`
- `2010`

**Примеры строк, которые не должны находиться:**
- `1999`
- `2011`
- `200`
- `20005`
- `22000`

*2. Номера карт Maestro*

**Описание задачи:** Найти номера карт платежной системы Maestro. Допустимые номера: начинаются с 50, 56–58 или 6, общая длина от 12 до 19 цифр.

**Регулярное выражение:**
```
\b(50[0-9]{10,17}|5[6-8][0-9]{10,17}|6[0-9]{11,18})\b
```

**Пояснение обозначений:**
- `\b` – граница слова.
- `50[0-9]{10,17}` – номера, начинающиеся с 50, за которыми следует от 10 до 17 цифр (всего 12–19 цифр).
- `5[6-8][0-9]{10,17}` – номера, начинающиеся с 56, 57 или 58.
- `6[0-9]{11,18}` – номера, начинающиеся с 6, за которыми следует от 11 до 18 цифр (всего 12–19 цифр).

**Примеры строк, которые должны находиться:**
- `5012345678901234` (16 цифр)
- `561234567890123456` (18 цифр)
- `571234567890123` (15 цифр)
- `58123456789012345` (17 цифр)
- `612345678901234567` (18 цифр)

**Примеры строк, которые не должны находиться:**
- `4912345678901234` (Visa)
- `5112345678901234` (MasterCard)
- `56123456789012345678` (20 цифр)
- `50123456789` (11 цифр)
- `6` (слишком коротко)

*3. IP-адрес с маской подсети*

**Описание задачи:** Найти IPv4-адреса с указанием маски подсети в формате `xxx.xxx.xxx.xxx/yy`, где `xxx` – число от 0 до 255, `yy` – число от 0 до 32.

**Регулярное выражение:**
```
\b((25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)\.){3}(25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)\/([0-9]|[12][0-9]|3[0-2])\b
```

**Пояснение обозначений:**
- `\b` – граница слова.
- `(25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)` – октет от 0 до 255.
- `\.` – точка (экранирована).
- `{3}` – три повторения октета с точкой.
- `\/` – символ '/' (экранирован).
- `([0-9]|[12][0-9]|3[0-2])` – маска от 0 до 32.

**Примеры строк, которые должны находиться:**
- `192.168.1.1/24`
- `10.0.0.1/8`
- `255.255.255.255/32`
- `0.0.0.0/0`
- `172.16.0.1/16`

**Примеры строк, которые не должны находиться:**
- `256.0.0.1/24` (октет >255)
- `192.168.1.1/33` (маска >32)
- `192.168.1.1.1/24` (лишний октет)
- `192.168.1/24` (не хватает октета)
- `192.168.1.1/` (нет маски)

# Лабораторная работа 5: Семантический анализатор для объявления комплексных чисел в Rust

**Цель работы**
Развить ранее созданный синтаксический анализатор (парсер) до семантического: построить абстрактное синтаксическое дерево (AST) и реализовать проверку контекстно-зависимых условий в соответствии с индивидуальным вариантом курсовой работы.

**Вариант задания**
**Тема:** Объявление комплексного числа с инициализацией.

**Пример верной строки:**
```rust
let complex_num2 = num::complex::Complex::new(3.1, -4.2);
```

**Дополнительные примеры корректных строк:**
- `let x = num::complex::Complex::new(1, 2);`
- `let y = num::complex::Complex::new(-5.0, 0.0);`

**Контекстно-зависимые условия (семантические правила)**

**1. Уникальность идентификаторов в области видимости**
- В одной области видимости нельзя объявлять два идентификатора с одинаковым именем.
- **Пример ошибки:**
  ```rust
  let x = num::complex::Complex::new(1.0, 2.0);
  let x = num::complex::Complex::new(3.0, 4.0);  // Ошибка: повторное объявление
  ```
- **Ожидаемое сообщение:** `Идентификатор 'x' уже объявлен ранее`


**2.Совместимость типов аргументов**
- Аргументы конструктора `Complex::new` должны быть числовыми литералами (целыми или вещественными).
- **Пример ошибки:**
  ```rust
  let z = num::complex::Complex::new("text", 5.0);  // Ошибка: нечисловой аргумент
  ```
- **Ожидаемое сообщение:** `Аргумент должен быть числовым литералом`

  **3. Допустимые значения числовых литералов**
- Целочисленные литералы должны находиться в диапазоне `i32` (от -2 147 483 648 до 2 147 483 647).
- Вещественные литералы должны находиться в диапазоне `f64` (примерно ±1.8e308).
- **Пример ошибки:**
  ```rust
  let big = num::complex::Complex::new(9999999999, 0.0);  // Ошибка: превышение i32
  ```
- **Ожидаемое сообщение:** `Целочисленный литерал 9999999999 выходит за пределы i32`

  **4. Использование объявленных идентификаторов**
- Путь `num::complex::Complex::new` считается предопределённым; другие идентификаторы в пути должны быть проверены на существование.
- **Пример ошибки:**
  ```rust
  let w = wrong::path::new(1.0, 2.0);  // Ошибка: неизвестный модуль
  ```
- **Ожидаемое сообщение:** `Идентификатор 'wrong' не объявлен и не является предопределённым`

  *Структура AST*

**Типы узлов (перечисление `NodeType`)**
- `LET_DECL` – объявление переменной (`let`).
- `CALL_EXPR` – вызов функции (конструктора).
- `PATH` – путь из идентификаторов (`num::complex::Complex::new`).
- `INT_LITERAL` – целочисленный литерал.
- `FLOAT_LITERAL` – вещественный литерал.
- `UNARY_EXPR` – унарное выражение (например, `-4.2`).

**Классы узлов (наследуются от `ASTNode`)**
- `LetDeclNode` – содержит поля `name` (имя переменной) и `value` (выражение инициализации).
- `CallExprNode` – содержит поля `callee` (путь) и `arguments` (список аргументов).
- `PathNode` – содержит поле `segments` (список строк идентификаторов).
- `IntLiteralNode` – содержит поле `value` (целое число).
- `FloatLiteralNode` – содержит поле `value` (вещественное число).
- `UnaryExprNode` – содержит поля `operator` (строка `"-"`) и `operand` (узел литерала).

**Инструкция по запуску**

Требования
- Python 3.8 или выше
- Установленные зависимости: `PySide6`

Установка зависимостей
```bash
pip install -r requirements.txt
```

**Запуск программы**
```bash
python main.py
```

**Использование**
1. Откроется главное окно редактора.
2. Введите код объявления комплексного числа в текстовую область.
3. Нажмите кнопку **«Пуск»** (или клавишу **F5**) для запуска анализа.
4. Результаты отобразятся на вкладках:
   - **«Лексемы»** – таблица токенов.
   - **«Ошибки»** – список лексических, синтаксических и семантических ошибок.
   - **«AST»** – текстовое представление дерева.
5. Для графической визуализации AST нажмите кнопку **«Показать AST»** (или клавишу **F7**).

**Запуск тестов**
```bash
python test_semantic_fixed.py
```
